# BJJ.Score — 프로젝트 아키텍처 문서

> **버전**: 2026.03.27  
> **앱명**: BJJ.Score (Brazilian Jiu-Jitsu Scoreboard)  
> **파일**: `bjj-scoreboard.html` (단일 HTML SPA)  
> **타겟**: 데스크톱 16:9 전용 (모바일은 별도 프로젝트로 진행 예정)  
> **외부 의존성**: Google Fonts CDN만 사용, 그 외 완전 오프라인 동작

---

## 1. 프로젝트 개요

주짓수(BJJ) 시합용/학습용 전자 점수판 웹앱. 대회장, 도장, 동아리에서 노트북으로 사용.
IBJJF(국제브라질리안주짓수연맹) 공식 규칙 기반 점수 체계 적용.

### 핵심 특징
- 단일 HTML 파일 SPA (HTML + CSS + JS 인라인)
- 서버 통신 없음, 완전 오프라인 동작
- 점수 입력: 좌클릭 +, 우클릭 -
- 실시간 타이머 (밀리초 정밀도)
- IBJJF 패널티 규칙 자동 적용
- Score Log 타임라인 기록

## 2. 화면 구조 (네비게이션 스택)

```
홈(#home)
  ├→ 시합용 점수판 타입 선택(#tsel)
  │    ├→ Basic 점수판(#bA)     ← 좌클릭+1, 우클릭-1
  │    └→ Detailed 점수판(#bC)  ← 2·3·4점 항목별 셀 클릭
  └→ 학습용 점수판(#trbd)       ← 기술별 버튼 (테이크다운/스윕/...)
```

### 화면 전환
- `goS(id)`: `.scr` 클래스 토글로 화면 전환
- 뒤로가기: `backSel()` (타입선택으로), `backHome()` (홈으로)
- 양쪽 모두 타이머 정지 + 점수 리셋 후 이동

## 3. 각 화면 상세

### 3.1 홈 (#home)

**레이아웃**: `pg-top`(상단바) + `pg-body`(중앙 콘텐츠)

**구성 요소**:
- 앱 제목: "BJJ.Score" (Orbitron, 검정 채우기 + 빨간 네온 외곽선)
- 부제: "Brazilian Jiu-Jitsu Scoreboard" (Noto Sans KR)
- 2개 선택 카드: 시합용 점수판 / 학습용 점수판

**조명 이펙트** (홈 화면 전용):
- 상단 설정 버튼이 조명 형태 (`.home-lamp`)
- 카드 호버 시: 조명→카드 방향 노란 빛줄기 (clip-path polygon + blur)
- 빈 공간 마우스 이동: 좁은 빛이 커서를 따라감
- 카드 진입/이탈: CSS clip-path transition으로 부드럽게 모핑
- 카드 호버 시 `scale(1.06)` + 노란 네온 테두리 + 동적 그림자

### 3.2 타입 선택 (#tsel)

**레이아웃**: `pg-top`(홈/뒤로/설정) + `pg-body`(제목 + 2개 카드)

**구성 요소**:
- 제목: "Score Board Type" (Orbitron, 빨간 네온)
- Basic 카드: 미니 프리뷰 테이블 포함
- Detailed 카드: 미니 프리뷰 테이블 포함 (2·3·4점 열)

**프리뷰 점수 (반드시 합산 일치)**:
- Basic: 선수1=6점(A:2,P:0), 선수2=7점(A:1,P:0)
- Detailed: 선수1→1×2+0×3+1×4=6, 선수2→2×2+1×3+0×4=7

### 3.3 Basic 점수판 (#bA)

**3-stack 레이아웃** (위→아래):
```
┌─────────────────────────────────┐
│ 상단바 (hdr)                     │ ← 고정 높이
├─────────────────────────────────┤
│ PLAYER │ TOTAL │  A  │  P      │ ← st-hdr
│ 선수 1  │  0    │  0  │  0     │ ← flex:1 (큰 영역)
│ 선수 2  │  0    │  0  │  0     │
├─────────────────────────────────┤
│           5:00                  │ ← tms (35% 높이)
└─────────────────────────────────┘
```

**그리드**: `.gA` = `3fr 1.5fr .8fr .8fr`
**조작**: 점수(TOTAL) 셀 좌클릭 +1, 우클릭 -1
**열 순서**: PLAYER → TOTAL → A(어드밴티지) → P(패널티)

### 3.4 Detailed 점수판 (#bC)

**그리드**: `.gC` = `2.8fr .8fr .8fr .8fr 1fr .7fr .7fr`
**열**: PLAYER → 2점 → 3점 → 4점 → TOTAL → A → P

**조작**:
- 2/3/4점 셀: 좌클릭 해당 점수 추가, 우클릭 해당 점수 차감
- TOTAL: 자동 계산 (cursor:default, 클릭 불가)
- A/P: 좌클릭 +1, 우클릭 -1

**2·3·4점 셀 스타일**: 흰색 배경(`#fff`), 작은 폰트(`clamp(40px,9vw,110px)`)
**TOTAL 셀 스타일**: 연초록 배경(`--sc-bg`), 큰 폰트

### 3.5 학습용 점수판 (#trbd)

**3-stack 레이아웃** (위→아래):
```
┌──────────────────────────────────────────────────────┐
│ 상단바 (hdr)                                          │
├──────────┬────┬────┬────┬────┬────┬────┬────┬────────┤
│ 선수 1    │ 2  │ 2  │ 2  │ 3  │ 4  │ 점수│ A  │ P    │ ← 44%
├──────────┼────┼────┼────┼────┼────┼────┼────┼────────┤
│ *3초 유지 │테이크│스윕│니온벨│가드패│마운트│점수│어드밴│패널티 │ ← 12%
├──────────┼────┼────┼────┼────┼────┼────┼────┼────────┤
│ 선수 2    │ 2  │ 2  │ 2  │ 3  │ 4  │ 점수│ A  │ P    │ ← 44%
├──────────────────────────────────────────────────────┤
│                    5:00                               │ ← tms (35%)
└──────────────────────────────────────────────────────┘
```

**그리드**: `2.2fr repeat(5,.6fr) 1.4fr .8fr .8fr`
**기술 버튼**: 흰색 배경, 검정 글씨, 고정 점수값 표시
**기술 → 점수 매핑**:
| 기술 | 점수 |
|------|------|
| 테이크다운 (td) | 2점 |
| 스윕 (sw) | 2점 |
| 니온벨리 (kob) | 2점 |
| 가드패스 (gp) | 3점 |
| 마운트/백 (mtbk) | 4점 |

**TOTAL 자동 계산**: `td×2 + sw×2 + kob×2 + gp×3 + mtbk×4 + bonus`
**bonus**: 상대 패널티 누적으로 인한 보너스 점수 (별도 관리)

**시간 설정 (오버레이 팝업)**:
- 분 프리셋: [4][5][6][7][8][10]
- 커스텀: 분/초 셀렉트 + SET
- IBJJF 벨트별: 화이트(5분), 블루(6분), 퍼플(7분), 브라운(8분), 블랙(10분)

## 5. 타이머 시스템

### 변수
```javascript
MD = 'T'|'TR'      // 모드 (Tournament / Training)
TP = 'A'|'C'        // 타입 (Basic / Detailed)
TT = 300            // 설정된 총 시간 (초)
RM = 300            // 남은 시간 (초, 정수부)
msRM = 0            // 남은 밀리초
RUN = false         // 실행 중 여부
FIN = false         // 경기 종료 여부
```

### 동작
- 타이머 영역 클릭 = 재생/정지 (`tog()`)
- `sta()`: `performance.now()` 기반 정밀 카운트다운, `setInterval` 50ms
- `pau()`: 정지 + 사운드
- 0초 도달 시 자동 정지 + 종료 사운드

### 표시
- 진행 중: 흰색
- 정지: 빨간색 (`.stopped`)
- 10초 이하: 깜빡이는 빨간색 (`.crit`)
- 0초: 노란색 (`.warn`)

### 소수점 표시 (설정)
- 없음 / 0.1초 / 0.01초
- 표시 시점: 항상 또는 N분 N초 이하일 때만
- 10분(600초) 이상 시 소수점 강제 비활성
- `_tw()` 함수: 각 문자를 고정폭 span으로 감싸 떨림 방지

### 사운드
- `AudioContext` + OscillatorNode (square wave)
- 시작: 3000Hz, 0.8초
- 정지: 2200Hz, 0.4초
- 종료: 3000Hz × 3회 반복
- 볼륨: `SET.vol` (0~1)

### 시간 편집
- 시계 아이콘 클릭 → 편집 패널 표시
- 프리셋: [4][5][6][7][8][10]분
- 커스텀: 분/초 셀렉트 + SET 버튼
- Training 전용: IBJJF 벨트별 버튼 (오버레이 팝업)

## 6. 점수 시스템

### 데이터 구조
```javascript
S = {
  1: { p2:0, p3:0, p4:0, t:0, pe:0, ad:0, bonus:0 },
  2: { p2:0, p3:0, p4:0, t:0, pe:0, ad:0, bonus:0 }
}
// p2/p3/p4: Type C 전용 (2점/3점/4점 횟수)
// t: 총점, pe: 패널티, ad: 어드밴티지
// bonus: 상대 패널티 누적으로 인한 보너스 점수

TRC = {  // Training 전용
  1: { td:0, sw:0, kob:0, gp:0, mtbk:0 },
  2: { td:0, sw:0, kob:0, gp:0, mtbk:0 }
}
```

### 셀 색상 (모든 점수판 공통)
| 셀 | 배경 | CSS 변수 |
|----|------|----------|
| P1 이름 | 흰색 | `--p1-bg: #ffffff` |
| P2 이름 | 하늘파랑 | `--p2-bg: #5c9dc5` |
| 점수/TOTAL | 연초록 | `--sc-bg: #b9f6ca` |
| 어드밴티지 | 진노랑 | `--adv-bg: #f9a825` |
| 패널티 | 빨강 | `--pen-bg: #d32f2f` |
| 글씨색 (전체) | 검정 | `--score-color: #111` |

**예외**: Type C의 2·3·4점 셀은 흰색 배경(`#fff`)

### 열 순서 (전 점수판 통일)
`이름 → [점수항목] → TOTAL → 어드밴티지(A) → 패널티(P)`

## 7. IBJJF 패널티 규칙

**모든 점수판(Basic/Detailed/Training)에 동일 적용.**

| 패널티 수 | 효과 | 로그 기록 |
|-----------|------|-----------|
| 2개 | 상대 어드밴티지 +1 | `A +1 (패널티 누적)` |
| 3개 | 상대 점수 +2 | `+2 (패널티 누적)` |
| 4개 | **실격(DQ)** | `실격 (DQ)` |

### 패널티 최대치
- 값은 **4가 최대** (`chkPen()`에서 `if(S[p].pe>4) S[p].pe=4`)
- 연속 클릭으로 4 초과 불가

### 실격 확인 팝업
- 패널티 4 도달 시 즉시 실격이 아닌 **확인 팝업** 표시
- "실격이 확실합니까?" + [실격 판정] [취소]
- 타이머는 계속 진행 (경기 중 판단 가능)
- 취소 시 패널티 -1 복원
- ESC로도 취소 가능

### 실격 화면
```
┌─────────────────────────────────┐
│            실격                  │ ← Noto Sans KR, 노란색, 초대형
│        패널티 4개 누적            │ ← 노란색
│                                 │
│ ┌────────┬──────┬──────┬───┬───┐│
│ │ 선수 1  │      │  2   │ 0 │ 1 ││ ← P1 흰배경
│ ├────────┼──────┼──────┼───┼───┤│
│ │ 선수 2  │ 실격  │  0   │ 4 │ 0 ││ ← P2 파란배경
│ └────────┴──────┴──────┴───┴───┘│
│                                 │
│   [🏠]  [← 돌아가기]  [▶ 다음경기] │
└─────────────────────────────────┘
```
- 실격 셀: 검정 배경, 빨간 글씨(`#ff1a1a`), font-weight:400
- 승자 셀: 비어있음 (텍스트 없음)
- 돌아가기: FIN 해제, 점수판 복귀
- 다음 경기: 리셋 후 새 경기

## 8. Score Log 시스템

### 구조
```javascript
LOG = [
  { time: "0:30", elapsed: 30, player: "양승환", pNum: 1, desc: "+2" },
  { time: "0:32", elapsed: 32, player: "양승환", pNum: 1, desc: "A +1" },
  ...
]
```

### 로그 합산 (Tournament 모드 전용)
- 같은 선수의 연속 점수 기록을 합산
- **조건**: 같은 선수(`pNum` 일치) + 순수 점수(`+N`/`-N` 형식) + 첫 기록 시점에서 2초 이내
- Training 모드에서는 합산 없음 (기술별로 이미 구분됨)
- `mergeLog()` 함수에서 처리

### 로그 창 레이아웃
```
┌─────────────────────────────────────┐
│              Score Log        [✕]   │
├─────────────────┬───────────────────┤
│ 타임라인 로그     │ 미니 점수판        │
│ (최신순 정렬)     │ (그리드 형식)      │
│ 0:30 양승환 +3   │ ┌───┬──┬──┬──┐   │
│ 0:14 선수2 +2    │ │P1 │6 │0 │2│   │
│                  │ │P2 │3 │1 │0│   │
│                  │ └───┴──┴──┴──┘   │
└─────────────────┴───────────────────┘
```

## 9. 설정 패널 (전 페이지 공통)

### 사운드
- 볼륨 슬라이더 (0~100%)
- 스피커 아이콘 토글 (ON=초록, OFF=빨강)
- 볼륨 변경 시 테스트 사운드 재생

### 타이머 소수점
- 없음 / 0.1초 / 0.01초
- 표시 시점: [항시] 또는 [N분 N초 이하]
- 항시 선택 시 안내: "10분 이상에서는 가독성을 위해 소수점이 표시되지 않습니다"

## 11. Reset / Undo 시스템

### Reset
- 타이머 정지
- 현재 상태 스냅샷 저장 (`_snap`)
- 점수/로그/타이머 모두 초기화
- Undo 버튼 활성화

### Undo (Reset 취소)
- `_snap`에서 상태 복원 (점수, 로그, 타이머, FIN 상태)
- 스냅샷 소비 후 Undo 버튼 비활성화

## 19. 파일 수정 시 주의사항

1. **단일 파일 구조**: HTML, CSS, JS가 모두 한 파일에 있음. 검색으로 찾아 수정할 것.
2. **인라인 이벤트 핸들러**: `onclick="cCk(1,1)"` 등. 분리하지 말 것.
3. **div 밸런스**: 수정 후 반드시 `<div>` 열기/닫기 수 확인.
4. **열 순서**: 모든 점수판에서 `이름 → 점수 → A → P` 순서 통일 유지.
5. **셀 색상**: CSS 변수로 관리. 직접 색상값 하드코딩 금지.
6. **폰트**: 용도별 지정된 폰트 사용. 임의 변경 금지.
7. **패널티 로직**: `chkPen()` 함수 하나로 모든 점수판 공통 처리. 개별 수정 금지.
8. **Training TOTAL**: `refTR()`에서 `TRC 합산 + bonus`로 계산. S[p].t 직접 수정 금지.
9. **타이머 정밀도**: `performance.now()` 기반. `Date.now()` 사용 금지.
10. **사운드**: `AudioContext` + `OscillatorNode`. HTML5 Audio 태그 사용 금지.
