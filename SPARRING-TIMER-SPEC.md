# 스파링 타이머 기능 추가 기획서

> **작성일**: 2026.04.01  
> **대상 파일**: `bjj-scoreboard.html` (기존 단일 SPA에 통합)  
> **우선순위**: 홈 화면 세 번째 메뉴로 추가  
> **핵심 원칙**: 기존 코드 구조·스타일·사운드 시스템 100% 재활용, 신규 의존성 없음

---

## 1. 배경 및 목적

주짓수 도장에서 스파링 시간 관리에 사용하는 인터벌 타이머들이 제각각이고 UI가 비직관적인 상황. 점수판 앱에 스파링 타이머를 통합하면 지도자가 하나의 앱으로 **시합 · 학습 · 스파링**을 모두 커버할 수 있어 앱의 매력도가 크게 상승.

### 핵심 사용 시나리오
- 관장/지도자가 노트북을 도장 한쪽에 설치
- 스파링 시간/휴식/라운드 수 설정 후 START
- 전체화면 타이머가 멀리서도 잘 보이게 표시
- 라운드 종료 → 자동 휴식 → 자동 다음 라운드 진행
- 종료 전 경고음으로 수련자들에게 알림

## 2. 네비게이션 변경

### 기존
```
홈(#home) — 2개 카드
  ├→ 시합용 점수판(#tsel → #bA/#bC)
  └→ 학습용 점수판(#trbd)
```

### 변경 후
```
홈(#home) — 3개 카드
  ├→ 시합용 점수판(#tsel → #bA/#bC)
  ├→ 학습용 점수판(#trbd)
  └→ 스파링 타이머(#sparSet → #sparRun)
       ├→ 설정 화면(#sparSet)  ← 파라미터 세팅
       └→ 실행 화면(#sparRun)  ← 전체화면 타이머
```

### 홈 화면 수정사항
- `.hcards` 안에 세 번째 카드 추가 (class: `hcard hcs`)
- 3개 카드 레이아웃이므로 gap/padding 축소 필요 (카드 3개가 균등하게 들어가야 함)
- 카드 아이콘: 시계+재생 조합 SVG
- 카드 텍스트: "스파링 타이머" / "라운드 · 휴식 · 인터벌\n수련 시간 관리"
- 카드 호버 이펙트: 기존 `.hct`, `.hcr`과 동일 (노란 네온)
- 홈 조명 빔 이펙트: 세 번째 카드에도 동일하게 적용 (`_hcards` querySelectorAll 범위 자동 확장)
- 클릭 시 `goS('sparSet')`

```

│  ┌──────────────────┐ ┌──────────────────┐   │
│  │ ≡ 라운드 수       │ │ ⚠ 종료 전 경고음  │   │
│  │ [−]  5  [+]      │ │ [없음][10초][30초]│   │
│  └──────────────────┘ └──────────────────┘   │

│  ┌────────────────────────────────────────┐   │
│  │ 💾 커스텀 프리셋                         │   │

│                                               │
│         5분 × 5R / 휴식 1분 / 총 29분         │ ← 요약 텍스트
```

### 3.2 그리드 구조
- 2열 그리드 (반응형: 640px 이하 1열)
- 각 섹션: 다크 패널 + border-radius + padding
- 커스텀 프리셋과 START 버튼: `grid-column: 1 / -1` (풀 width)

### 3.3 파라미터 상세

| 항목 | 변수명 | 기본값 | 범위 | UI |
|------|--------|--------|------|-----|
| 라운드 시간 | `SP.rndTime` | 300 (5분) | 30초~1200초 | 프리셋 버튼 + 분/초 커스텀 |
| 휴식 시간 | `SP.restTime` | 60 (1분) | 10초~300초 | 프리셋 버튼 + 분/초 커스텀 |
| 라운드 수 | `SP.rounds` | 5 | 1~30 | ±버튼 |
| 경고음 | `SP.warn` | 10 (10초 전) | 0/10/30 | 토글 버튼 |

### 3.4 프리셋 버튼 동작
- 클릭 시 해당 값 적용 + `.on` 클래스 토글 (초록 border + 초록 텍스트)
- 커스텀 값 적용 시 프리셋 버튼의 `.on` 해제
- 프리셋 배열:
  - 라운드 시간: `[3분, 4분, 5분, 6분, 7분, 8분, 10분]`
  - 휴식 시간: `[30초, 45초, 1분, 1분30초, 2분]`
  - 경고음: `[없음, 10초, 30초]`

### 3.5 커스텀 프리셋 저장 시스템
- `localStorage` 키: `bjjSpPresets`
- 저장 형식: `[{name, rnd, rest, rounds, warn}, ...]`
- 최대 10개
- 저장: `prompt()`로 이름 입력 → 현재 설정 저장
- 불러오기: 칩 클릭 → 전체 설정 반영
- 삭제: 칩 내 X 아이콘 클릭 → `confirm()` 후 삭제
- 칩 UI: 둥근 알약 형태, 호버 시 초록 테두리

### 3.6 요약 텍스트
- START 버튼 위에 표시
- 형식: `{라운드시간} × {라운드수}R / 휴식 {휴식시간} / 총 {총시간}`
- 설정 변경 시 실시간 업데이트
- 총 시간 계산: `rndTime × rounds + restTime × (rounds - 1)`

### 3.7 START 버튼
- 초록 배경, 검정 텍스트, 대형
- `box-shadow`로 초록 글로우
- 호버: 밝기 + 스케일 약간 증가
- 클릭 → `#sparRun` 화면으로 전환 + 타이머 초기화 (자동 시작하지 않음)

## 4. 실행 화면 (#sparRun)

### 4.1 레이아웃

```
┌──────────────────────────────────────────────┐
│ [←뒤로]          Round 1 / 5        [↻리셋]  │ ← 상단 바
├──────────────────────────────────────────────┤
│                                               │
│                  FIGHT                        │ ← 상태 (초록/노랑/빨강)
│                                               │
│                  4:32                          │ ← 초대형 타이머 숫자
│                                               │
│               Round 1                         │ ← 라운드 정보
│           ████████░░░░░░░░                    │ ← 진행률 바
│           총 남은시간: 24:32                    │
│                                               │
└──────────────────────────────────────────────┘
```

### 4.2 상단 바
- 왼쪽: 뒤로 버튼 (설정 화면으로, 타이머 정지)
- 가운데: `Round N / M` 텍스트
- 오른쪽: 리셋 버튼 (라운드 1부터 다시)

### 4.3 타이머 표시 영역 (전체 클릭 가능)
- **전체 영역 클릭 = 시작/정지** (기존 점수판 타이머와 동일 UX)
- Space 키로도 시작/정지

#### 상태 텍스트 (`#spState`)
| 상태 | 텍스트 | 색상 | CSS class |
|------|--------|------|-----------|
| 대기 | "준비" | 회색 | `.ready` |
| 스파링 진행 | "FIGHT" | 초록 | `.fight` |
| 정지 (일시정지) | "정지" | 회색 | `.ready` |
| 휴식 | "REST" | 노란색 | `.rest` |
| 완료 | "완료" | 빨간색 | `.done` |

#### 타이머 숫자 (`#spTimer`)
- 폰트: Teko (기존 타이머와 동일)
- 크기: `clamp(140px, 42vw, 480px)` — 기존 점수판 타이머보다 더 크게
- `_tw()` 함수 재사용하여 글자 떨림 방지
- 상태별 색상:
  - FIGHT 진행 중: 흰색 + 초록 text-shadow
  - REST: 노란색 + 노란 text-shadow
  - 경고 (warn초 이하): 빨간색 + 깜빡임 애니메이션
  - 일시정지: 빨간색 (기존 `.stopped`과 동일 컨셉)
  - 완료: 빨간색

#### 라운드 정보 (`#spRoundInfo`)
- 진행 중: "Round N"
- 휴식 중: "다음 라운드 준비"
- 완료: "전체 NR 완료"
- 대기: "클릭하여 시작"

#### 진행률 바
- 현재 라운드/휴식 내 진행률
- FIGHT: 초록 바
- REST: 노란 바

#### 총 남은시간
- 전체 세션 남은 시간 (모든 라운드 + 휴식 합산)
- 회색 텍스트, 하단 표시

## 5. 타이머 동작 로직

### 5.1 상태 변수

```javascript
var SP = {
  rndTime: 300,    // 라운드 시간 설정 (초)
  restTime: 60,    // 휴식 시간 설정 (초)
  rounds: 5,       // 총 라운드 수
  warn: 10,        // 경고음 시점 (초), 0이면 비활성
  curRound: 1,     // 현재 라운드 번호
  phase: 'idle',   // 'idle' | 'fight' | 'rest' | 'done'
  rndRM: 0,        // 현재 페이즈 남은 시간 (초, 정수부)
  msRM2: 0,        // 남은 밀리초
  spRUN: false,    // 실행 중 여부
  spIV: null,      // setInterval 핸들
  spLK: null       // performance.now() 마지막 체크
};
```

### 5.2 타이머 엔진
- **기존 타이머 엔진과 동일 패턴** 사용
- `performance.now()` 기반, `setInterval` 50ms
- 별도 변수/함수 사용 (기존 `RM`, `RUN`, `IV` 등과 충돌 방지)

### 5.3 페이즈 전환 흐름

```
[START 클릭] → phase='fight', rndRM=rndTime
    ↓ (클릭으로 시작)
[FIGHT 카운트다운]
    ↓ (0초 도달)
[라운드 종료 사운드] → 마지막 라운드면 → phase='done' (종료)
    ↓ (마지막 아니면)
phase='rest', rndRM=restTime → 1.5초 후 자동 시작
    ↓
[REST 카운트다운]
    ↓ (0초 도달)
[휴식 종료 사운드] → curRound++, phase='fight', rndRM=rndTime → 1초 후 자동 시작
    ↓
(반복)
```

### 5.4 자동 전환 타이밍
- 라운드 종료 → 휴식 시작: **1.5초 딜레이** (종료 사운드 인지 시간)
- 휴식 종료 → 다음 라운드 시작: **1초 딜레이**
- 딜레이 중 클릭 시: 수동 시작/정지 가능

### 5.5 일시정지
- 클릭 또는 Space로 언제든 정지/재개 가능
- 정지 시 정지 사운드 재생 (기존 `sndStop()` 재사용)
- phase='done' 상태에서는 클릭 무시

### 5.6 리셋
- 라운드 1, phase='fight'로 초기화
- 타이머 정지, 자동 시작하지 않음

## 6. 사운드 체계

기존 `AudioContext` + `OscillatorNode` 시스템 100% 재사용.

| 이벤트 | 사운드 | 구현 |
|--------|--------|------|
| 타이머 시작/재개 | 3000Hz, 0.8초 | 기존 `sndStart()` 재사용 |
| 타이머 정지 | 2200Hz, 0.4초 | 기존 `sndStop()` 재사용 |
| 라운드 종료 | 3000Hz × 3연속 | 기존 `sndEnd()` 재사용 |
| 경고음 (N초 전) | 1500Hz, 0.2초 × 2회 | 신규 `spSndWarn()` |
| 휴식 종료 | 2000Hz, 0.5초 | 신규 `spSndRest()` |

### 경고음 트리거 조건
- `SP.warn > 0` 이고
- `SP.rndRM === SP.warn` 도달 시점에 1회만 발생
- 휴식 중에는 경고음 없음 (스파링 라운드에서만)

## 9. 총 남은시간 계산

```
총 시간 = rndTime × rounds + restTime × (rounds - 1)

경과 시간 = (curRound - 1) × (rndTime + restTime)
          + (현재 페이즈 총 시간 - rndRM)

남은 시간 = 총 시간 - 경과 시간
```

## 11. 구현 시 주의사항

1. **기존 타이머와 변수 분리**: 기존 `RM`, `RUN`, `IV`, `LK` 등과 겹치지 않도록 `SP` 객체 안에 별도 관리
2. **기존 `tog()`, `sta()`, `pau()` 호출 금지**: 스파링 타이머 전용 함수 (`spTog`, `spStart`, `spStop`) 사용
3. **사운드 함수는 재사용**: `sndStart()`, `sndStop()`, `sndEnd()`, `beepFlat()` 등 기존 함수 그대로 사용
4. **`_tw()` 함수 재사용**: 타이머 숫자 떨림 방지
5. **`fmt()` 함수 재사용**: 시간 포맷팅
6. **div 밸런스 확인**: 수정 후 반드시 `<div>` 열기/닫기 수 확인
7. **ESC 핸들러 분기 추가**: `#sparSet`, `#sparRun` 화면 추가
8. **홈 화면 빔 이펙트**: `_hcards` querySelectorAll 범위가 자동으로 세 번째 카드까지 포함되는지 확인
9. **3개 카드 레이아웃**: `.hcards` gap/max-width 조정, 카드 padding 축소
10. **설정 패널(⚙)은 기존 것 공유**: `openSettings()` 그대로 사용
